# DM - Particle Diffusion Model Pipeline

This folder contains a unified particle generation pipeline using Denoising Diffusion Probabilistic Models (DDPM) with a cosine noise schedule and charge balance loss.

## Overview

The pipeline implements a diffusion-based generative model for simulating particle physics events. It can:
- **Train** a diffusion model on real event data
- **Sample** synthetic events from a trained model
- **Evaluate** the quality of generated samples

Raw Guinea-Pig data: `/ceph/submit/data/group/fcc/ee/beam_backgrounds/guineapig/FCCee_Z_GHC_V25p3_4_FCCee_Z256_2T_grids8/*.pairs` and `/ceph/submit/data/user/b/bmaier/BiB/*.pairs`. Use output_XXX.pairs only.

Data for training and sampling available: `/ceph/submit/data/user/h/haoyun22/dm_generated_results/guineapig_raw_trimmed_new.npy`

Generated events, model check point, loss history, meta data, 2D 3D classifier data of `pipeline_cosine_charge.py` available in `/ceph/submit/data/user/h/haoyun22/dm_generated_results/results_nosquash_cosine_chargeloss/`

## Main Pipeline: `pipeline_cosine_charge.py`

The main script implements a complete DDPM-based particle generator with support for variable-length particle multiplicities and charge conservation constraints.

### Key Features

- **Cosine noise schedule**: More stable training than linear schedules
- **Charge balance loss**: Enforces e⁻-e⁺ balance per event
- **Batch sampling**: Efficient generation of multiple events
- **GPU monitoring**: Track resource utilization during training
- **Resume capability**: Continue training from checkpoints

## Usage

### 1. Training Mode

Train the diffusion model on real data:

```bash
python pipeline_cosine_charge.py train \
  --data_path /path/to/data.npy \
  --outdir ./results \
  --epochs 100 \
  --batch_size 32 \
  --max_particles 64 \
  --T 1000
```

**Parameters:**
Can also be modify in the class `CFG` of the pipeline.
- `--data_path`: Path to training data (`.npy` format with variable-length events), generated by `track_data.py`
- `--outdir`: Output directory for model checkpoints and metadata
- `--epochs`: Number of training epochs (default: 50)
- `--batch_size`: Batch size for training (default: 2)
- `--max_particles`: Maximum particles per event (default: 1300)
- `--T`: Number of diffusion steps (default: 1000)
- `--seed`: Random seed for reproducibility (default: 123)

**Output:**
- `ckpt_last.pt`: Latest model checkpoint
- `meta.pt`: Training metadata and model configuration
- `train_losses.npy`: Training loss history
- `val_losses.npy`: Validation loss history

### 2. Sampling Mode

Generate synthetic events from a trained model:

```bash
python pipeline_cosine_charge.py sample \
  --outdir ./results \
  --n_events 1000 \
  --sample_batch_size 32 \
  --num_steps 200
```

**Parameters:**
- `--outdir`: Directory containing the trained model
- `--n_events`: Number of events to generate (default: 1000)
- `--sample_batch_size`: Batch size for sampling (default: 16)
- `--num_steps`: Diffusion steps for sampling (default: None = use training T 1000)
  - Lower values → faster generation, lower quality
  - Higher values → slower generation, higher quality

**Output:**
- `generated_events_{num_steps}steps.npy`: Generated events array


## Supporting Scripts

Processes raw `.pairs` files from GuineaPig simulation into consolidated event arrays.

Trains classifiers to distinguish real vs generated events. Lower classifier AUC indicates better sample quality.

```bash
# Graph Neural Network evaluation

python classifier.py --run_gnn

# Ablation study for GNN
python classifier.py --gnn_ablation
```

- Fractional difference plots across sampling steps

## Data Format

### Input Format (Training Data)

Variable-length events with 7 features per particle:
```
Event 1: [E_signed1, betax1, betay1, betaz1, x1, y1, z1, ...]
Event 2: [E_signed2, betax2, betay2, betaz2, x2, y2, z2, ...]
...
```

Where:
- `E_signed`: Energy with sign indicating particle type (e⁻: +, e⁺: -)
- `betax, betay, betaz`: Normalized momentum vector (β = p/E)
- `x, y, z`: Vertex position

### Output Format (Generated Events)

Explicit PDG format:
```
[pdg, E, betax, betay, betaz, x, y, z]
```

Where:
- `pdg`: Particle ID (11 for e⁻, -11 for e⁺)
- Other fields: Same as input format

## Example Workflow

```bash
# 1. Prepare data
python track_data.py

# 2. Train model
python pipeline_cosine_charge.py train \
  --data_path guineapig_raw_trimmed_new_2.npy \
  --outdir results_cosine_charge \
  --epochs 50 \
  --batch_size 32

# 3. Generate samples at different inference steps
python pipeline_cosine_charge.py sample \
  --outdir results_cosine_charge \
  --n_events 10000 \
  --num_steps 100

python pipeline_cosine_charge.py sample \
  --outdir results_cosine_charge \
  --n_events 10000 \
  --num_steps 500

# 4. Evaluate quality
python classifier.py --data --steps 100
python classifier.py --run

python analysis.py
```
